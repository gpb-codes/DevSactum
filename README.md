## `> diagnose --problema`

```diff
- LinkedIn      →  Red profesional, pero sin contexto técnico real
- GitHub        →  Código, pero sin red ni oportunidades
- Slack/Discord →  Comunidad, pero efímera y sin trazabilidad
- Upwork        →  Clientes, pero sin reputación técnica verificable
- Twitter/X     →  Visibilidad, pero sin valor profesional concreto

+ DevSactum     →  Todo en un solo lugar, diseñado para developers
```

## `> ls features/`

<div align="center">

| Módulo | Descripción |
|---|---|
| **◈ Perfil técnico** | Registro vivo: stack, nivel verificado, repos integrados y reputación real |
| **◇ Sistema de reputación** | Se construye con proyectos, reviews y contribuciones — no se compra |
| **⟐ Proyectos colaborativos** | El núcleo. Roles definidos, stack declarado, resultado concreto |
| **◫ Bolsa de empleo** | Filtrada por stack y nivel. Sin spam, sin posiciones irrelevantes |
| **⊞ Portafolio integrado** | Conectado a GitHub y deployments. Se construye solo mientras trabajás |
| **◧ Comunidad técnica** | Rankeada por valor técnico, no por engagement. Sin ruido |

</div>
