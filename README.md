## ¿Qué es DevSactum?

Hoy los developers viven fragmentados entre cinco plataformas que no se hablan entre sí. Cinco perfiles. Cinco reputaciones. Cinco identidades. Todo para lograr lo que debería pasar en un solo lugar.

**DevSactum es ese lugar.** Un ecosistema cerrado y especializado donde desarrolladores, empresas y clientes conviven con reglas diseñadas específicamente para el mundo del software.

---

## El problema

```
LinkedIn      →  red profesional, pero sin contexto técnico real
GitHub        →  código, pero sin red ni oportunidades
Slack/Discord →  comunidad, pero efímera y sin trazabilidad
Upwork        →  clientes, pero sin reputación técnica verificable
Twitter/X     →  visibilidad, pero sin valor profesional concreto
──────────────────────────────────────────────────────────────────
DevSactum     →  todo en un solo lugar, diseñado para developers
```

---

## Funcionalidades

| # | Módulo | Descripción |
|---|--------|-------------|
| 01 | **Perfil técnico** | Registro vivo: stack, nivel verificado, repos integrados y reputación real |
| 02 | **Sistema de reputación** | Se construye con proyectos, reviews y contribuciones — no se compra |
| 03 | **Proyectos colaborativos** | Roles definidos, stack declarado, resultado concreto |
| 04 | **Bolsa de empleo** | Filtrada por stack y nivel. Sin spam, sin posiciones irrelevantes |
| 05 | **Portafolio integrado** | Conectado a GitHub y deployments. Se construye solo mientras trabajás |
| 06 | **Comunidad técnica** | Rankeada por valor técnico, no por engagement. Sin ruido |

## Instalación local

```bash
# Clonar el repositorio
git clone https://github.com/devsactum/devsactum.git
cd devsactum

# Levantar con Docker
docker compose up --build

# O instalar dependencias manualmente
npm install        # frontend (Next.js)
pip install -r requirements.txt  # backend (Python)
```
